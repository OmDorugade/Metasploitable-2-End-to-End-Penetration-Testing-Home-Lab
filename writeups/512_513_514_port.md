<div align="center">
This target exposes legacy R-services (rexec, rlogin, rsh) on ports 512, 513, and 514. These services rely on weak trust-based authentication, allowing unauthorized remote access when misconfigured.

## 1. R-Services Reconnaissance using Nmap

### 🔍 Command Used

nmap -p- -sV -sC <target-ip>

### 📌 Explanation

- `-p` → Specify ports  
- `-sV` → Detect service versions  

### 📊 Key Findings

- Port 512 → rexec  
- Port 513 → rlogin  
- Port 514 → rsh  

👉 Indicates legacy remote access services are enabled.

## 2. Understanding the Vulnerability

### 📌 Key Issue

R-services use **.rhosts** and **/etc/hosts.equiv** for authentication.

👉 If misconfigured:
- No password required  
- Trust-based login allowed  
- Remote root access possible
